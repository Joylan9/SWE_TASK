One-line project summary: This project is a polished wall calendar web component that combines a physical calendar look with interactive date-range selection, notes, theme switching, and responsive behavior.

1. What This Project Is

This project was built for a frontend engineering challenge. The goal was to turn a static wall calendar design into a working and responsive web component. I used React with TypeScript and focused only on the frontend, because the task clearly said there was no need for a backend or database.

My solution keeps the visual feeling of a physical wall calendar by using a large hero image, a spiral binding effect at the top, a clear month badge, and a compact notes area beside the calendar grid on desktop. On mobile, the layout becomes vertical so it is still easy to use on touch screens.

2. Main Features You Should Mention

Wall Calendar Look

Decorative spiral rings, a monthly hero image, a blue month-and-year badge, and a layout that feels like a real printed calendar.

Date Range Selection

Users can click a start date and an end date. The app highlights the start, the end, and the days in between. It also supports single-day selection.

Notes Section

Users can add notes either for the whole month or for a selected date range. Notes are stored in localStorage, so they stay after refresh.

Responsive and Interactive

The component works on desktop and mobile, supports touch interactions, keyboard navigation, theme switching, and animated month changes.

4. Quick Technical Summary

  • The app entry is very small. App.tsx only renders the main WallCalendar component.
  • WallCalendar.tsx is the main orchestrator. It connects the header, month navigator, notes panel, date grid, and theme toggle.
  • State is managed with a reducer inside useCalendar.ts, which makes month changes, date selection, and theme updates easier to control.
  • Date logic is handled with date-fns, which keeps the code clean and reliable.
  • Notes and theme preferences are persisted in localStorage.
  • The project includes automated tests with Vitest and React Testing Library.

6. What to Say About the Code Structure

Main Components

  • WallCalendar.tsx: main container and feature wiring.
  • CalendarHeader.tsx: spiral header, hero image, month badge, and page-flip animation.
  • MonthNavigator.tsx: previous and next controls plus year dropdown.
  • CalendarGrid.tsx: renders the weekly headers and the 42 calendar cells.
  • CalendarDay.tsx: individual day cell with selection state, note indicator, and holiday dot.
  • NotesPanel.tsx: note creation, note list, and note deletion.
  • ThemeToggle.tsx: light and dark mode button.

Custom Hooks

  • useCalendar.ts: reducer-driven state for month, year, selected range, notes, and theme.
  • useDateRange.ts: hover-preview logic for the date range.
  • useNotes.ts: add, delete, filter, and label notes based on the current month or selected range.

7. Simple Technical Explanation You Can Say

"From an architecture point of view, I tried to keep the component modular. The main WallCalendar component coordinates everything, while smaller components handle one responsibility each. I used custom hooks so the state logic is separated from the visual components. That makes the code easier to read, test, and maintain."

Date Range Logic

The date selection follows a simple state machine. The first click creates the start date. The second click creates the end date. If the second date is before the first one, the app swaps them automatically. If the user clicks again after the range is complete, the app resets and starts a new selection. This keeps the interaction easy to understand.

Notes Logic

The notes system stores note text, month, year, and optional range information. Because of that, notes can be shown in the notes panel and also reflected back into the calendar with note indicators. Since the task was frontend-only, localStorage was the best choice for lightweight persistence.

Styling Approach

Styling is handled with global CSS and design tokens using CSS custom properties. This helped keep the look consistent and made theme switching much easier. The design also uses clip-path for the hero image shape, subtle shadows, responsive layout changes, and print styles as a bonus.

Accessibility

I also paid attention to accessibility. The calendar uses semantic roles like grid and gridcell, the selected states use ARIA attributes, buttons have labels, and keyboard users can move across the date cells with arrow keys.

Performance

The calendar has many day cells, so I wrapped the day component with React.memo and used a custom comparison function. This helps avoid unnecessary re-renders when only a small part of the state changes.
